# Token Usage & Cost Calculation

## Overview

RunClaude reads Claude Code's JSONL log files and extracts token usage data in real time. This document explains how tokens are counted, how costs are estimated, and how data is aggregated at multiple time scales.

---

## 1. Data Source

Claude Code writes one JSONL file per workspace session to:

```
~/.claude/projects/<project-hash>/<session-id>.jsonl
```

Each line is a JSON event. `JSONLParser` scans each line for an `assistant`-type entry that contains a `usage` block.

---

## 2. Token Counting

Four token types are extracted from each `usage` block:

| Field | Description |
|---|---|
| `input_tokens` | Tokens sent in the prompt |
| `output_tokens` | Tokens generated by Claude |
| `cache_creation_input_tokens` | Tokens written to the prompt cache |
| `cache_read_input_tokens` | Tokens read back from the cache (cheaper) |

```
totalTokens = input + output + cacheCreation + cacheRead
```

Each record also carries the model name (e.g. `claude-sonnet-4-5`) and a timestamp, which are used for cost calculation and aggregation.

---

## 3. Cost Calculation

Costs are estimated using hardcoded per-million-token rates from `CostCalculator.swift`. The model is matched by name prefix.

### Pricing table (as of early 2026)

| Model | Input | Output | Cache write | Cache read |
|---|---|---|---|---|
| Claude Opus 4 | $15.00 | $75.00 | $18.75 | $1.50 |
| Claude Sonnet 4 / 3.5 | $3.00 | $15.00 | $3.75 | $0.30 |
| Claude Haiku | $0.80 | $4.00 | $1.00 | $0.08 |

Unknown models fall back to Sonnet-tier pricing.

### Formula

```
cost = (inputTokens   × inputRate
      + outputTokens  × outputRate
      + cacheCreation × cacheCreationRate
      + cacheRead     × cacheReadRate) / 1_000_000
```

Each token type is billed at its own rate because cache reads are significantly cheaper than fresh input tokens.

---

## 4. Aggregation Layers

`TokenAggregator` maintains three simultaneous layers of aggregation:

### 4.1 Sliding window (10 seconds)

Used to compute **tokens per second**, which drives the menu bar animation speed. Samples older than 10 s are pruned on every ingest.

```
tokensPerSecond = windowTokens / effectiveWindowDuration
```

### 4.2 Daily totals

Accumulates input, output, cache creation, and cache read tokens separately for the current calendar day. Resets at midnight. Broken down by model for per-model cost display.

### 4.3 Per-file sessions

One `FileSession` entry is maintained per JSONL file path (= one Claude Code workspace). Tracks:

- `firstRecord` / `lastRecord` timestamps
- `totalTokens`
- `modelBreakdown` — per-model token counts for accurate cost calculation

**Session cost** = sum of `CostCalculator.cost(for:)` across each model's usage breakdown.

**Burn rate** (tokens/min):
```
burnRate = totalTokens / elapsedMinutes
```

**Projected cost** (linear extrapolation to a full 8-hour session):
```
projectedCost = sessionCost × (480 / elapsedMinutes)
```

A session is considered **active** if its last token record arrived within the past 30 seconds. Sessions disappear from the Live tab after 5 minutes of inactivity.

---

## 5. Display Formatting

Costs are formatted by magnitude:

| Range | Format |
|---|---|
| < $0.01 | `$0.0042` (4 decimal places) |
| $0.01 – $0.99 | `$0.123` (3 decimal places) |
| ≥ $1.00 | `$1.23` (2 decimal places) |
